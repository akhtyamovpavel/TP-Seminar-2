# Мануал по использованию git

## Выкачивание или создание репозитория
```
git clone <url> <folder> -  склонировать репозиторий (при этом указывать папку совсем необязательно)
git init - создать репозиторий внутри папки
```

## Управление файлами
```
git status - узнать статус внутри бранча
git add ... -  добавить файлы (лучше не использовать опцию --all, пока не прописали .gitignore)
git rm ... - удалить файлы
git rm -r ... - удалить рекурсивно
git rm --cached ... - удалить файл из гита, оставив его на диске
git rm -f ... - удалить файл насовсем (из гита и из диска)
git mv <from> <to> - переместить файл
git cp <from> <to> - скопировать файл
```

## Коммиты и пуши
```
git commit -m <message> - создать коммит с сообщением
git remote add <remote name> <remote url> - добавить удаленный репозиторий
git push <remote name> <branch> - залить изменения в сервер git
```

## Статусы
```
git branch -vv - посмотреть список локальных веток
git remote -vv - посмотреть список подключенных репозиториев
git diff - посмотреть разность между изменениями в локальной ФС и локальным git-ом
git log - посмотреть список предыдущих коммитов
```

## Получение изменений
```
git fetch <remote name> <remote branch> - выкачать данные по ветке (если не указывать параметры, то выкачает по всем веткам внутри upstream repo - который с -u был при первом пуше)
git merge <branch> - слить ветку <branch> в текущую ветку
git reset - откат локальных изменений
git checkout - <file> - откат изменений для одного файла
git pull <remote name> <remote branch> - fetch + merge - получить изменения с сервера + слить с локальными изменениями
```
